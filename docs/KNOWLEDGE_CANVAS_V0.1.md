# 知识画布 v0.1

## 目标

知识画布用于表达“资料之间为什么有关”，而不是再次复制左侧文件夹结构或关系图谱。

首版支持把三类对象放在同一个无限画布中：

1. 文本卡片：记录问题、观点、结论和待验证假设。
2. 表格卡片：整理对比、证据、参数和任务清单。
3. 笔记卡片：引用知识库中的真实 Markdown 文档，显示摘要、标签和原路径。

## 核心工作流

- 从左侧工具栏进入“知识画布”。
- 在画布工具栏添加文本、表格或知识库笔记。
- 拖动空白区域平移，使用滚轮围绕鼠标位置缩放。
- 拖动卡片标题移动卡片，拖动右下角调整卡片尺寸。
- 选择一张卡片的连线按钮，再选择目标卡片建立关系。
- 关系可以标记为相关、支持、冲突、依赖或引用，并可补充自定义说明。
- 按住 Shift 或 Ctrl 选择多张卡片，再建立主题分组。
- Delete 删除所选卡片、连线或分组；Ctrl+Z 和 Ctrl+Shift+Z 撤销与重做。
- “适应全部内容”会把已有卡片和分组重新放入当前可视区域。

## 持久化

可写本地知识库会把画布保存为：

```text
<知识库根目录>/.knowledge-agent/canvas.json
```

画布文件只保存卡片位置、尺寸、表格内容、连线、分组和视口，不改写 Markdown 正文。

- Windows App 通过专用 Tauri 命令读写该文件。
- Web App 只在用户主动选择并授权本地文件夹后读写该文件。
- GitHub 公开库和只读磁盘结构模式不写入画布。
- 切换知识库前会先完成当前画布保存，避免旧画布写入新目录。

## 设计取法

首版吸收了几类成熟画布的共同做法：

- Obsidian Canvas：文件卡片、文本卡片、分组、颜色与连线。
- Miro：结构化卡片、可标注连接线、分组和快速整理。
- Heptabase：以卡片和白板组织研究材料，并保留深入原资料的入口。
- tldraw：把文档状态与临时会话状态分离，并将图形、绑定和视口作为可迁移数据保存。

## 当前边界

- v0.1 只有一张主画布，后续再增加多画布、画布文件树和画布间跳转。
- 笔记卡片引用真实文档，但暂不在卡片内直接编辑完整 Markdown。
- 表格是轻量数据表，不承担电子表格公式或复杂计算。
- 连线语义由用户确认；Agent 自动归纳和自动布板留到后续版本。
- 协作仍通过知识库文件和 Git 进行，未加入多人实时光标或同屏编辑。

## 下一轮观察重点

- 多张画布的命名、目录和跨画布引用。
- Agent 根据一组资料提出卡片、分组和连线，但必须保留证据与人工确认。
- 卡片快速复制、框选、对齐、吸附、层级和键盘工作流。
- 关系筛选与局部聚焦，避免大型画布的连线噪声。
- 将画布作为独立可分享对象，而不暴露整份私人知识库。
