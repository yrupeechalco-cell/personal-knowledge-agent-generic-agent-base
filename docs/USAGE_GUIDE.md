# 使用指南：从文件夹到日用知识库

这份指南对应当前公开版。它介绍已经可以实际使用的操作，并区分桌面端与浏览器预览的边界。

## 1. 选择知识库存储位置

点击左上角的文件夹图标，打开“存储空间”。

![存储空间入口](images/local-storage.png)

桌面 App 有两种真实本机操作：

1. **打开已有文件夹**：选择一个包含 Markdown 文件的普通文件夹。它不需要是 Obsidian vault；App 会读取其中允许范围内的 Markdown 文件并建立索引。
2. **选择位置并新建**：在任意磁盘位置创建一个新的知识库文件夹，再持续在这里存储笔记。

浏览器预览版受浏览器权限限制，只能在你手动授权后打开文件夹，不能直接在任意位置创建文件夹。日常读写请使用 Windows 桌面 App。

Web 版还可以只读打开 GitHub 公开仓库：

1. 在“存储空间”的“GitHub 公开知识库”中输入 `owner/repo` 或仓库首页网址。
2. 点击“打开”，页面会读取仓库默认分支中的安全 Markdown。
3. 文件树、双链、图谱、标签、阅读和 Agent 问答可以正常使用。
4. 新建、编辑、重命名、删除、应用 Agent diff 和写回操作全部禁用。

主项目保持零预置内容。官方演示库位于独立公开仓库，可点击“打开官方示例知识库”按需载入；普通主站网址仍从空白工作区启动。

公开仓库默认通过 GitHub API 自动发现文件。需要高频公开展示时，可以在仓库根目录提供 `.knowledge-agent-vault.json` 静态清单；官方示例库已经使用该方式，因此匿名 API 限流时仍可读取。清单只声明 Markdown 路径、字节大小和分支，不包含私钥或 GitHub 凭证。

## 2. 浏览和组织文件

左栏把磁盘文件夹按上下级展示。文件夹可折叠，筛选框可快速过滤笔记；大的文件夹和普通笔记会保留不同的信息层级。

对笔记或文件夹右键，可以执行：

- 在当前位置新建笔记
- 重命名
- 复制或剪切
- 删除到回收流程

顶部标签页不会因为打开下一篇笔记而丢失上一篇。可以随时点击已有标签回到原来的阅读位置。

## 3. 看关系图谱

![GitHub 公开示例库的全库知识地形](images/workspace-graph.png)

中间栏的关系图谱显示整个知识库中由 `[[笔记]]` 形成的连接：

- 左键拖动空白区域：平移整个图谱。
- 滚轮：围绕鼠标位置缩放。
- 拖动节点：调整单个节点位置；松手后图谱会继续自然分布。
- 悬停节点：无关节点变暗，相关节点与连线被强调，便于在大库中追踪局部关系。
- 单击节点：打开对应笔记。
- 缩小图谱：标题根据实际像素大小渐隐；不再可读时自动隐藏，放大后恢复。

图谱的关系来自文档内容，不是根据按钮或示例预先安排的。没有 `[[链接]]` 的文件不会凭空被画成强关系。

## 4. 读写笔记与局部关系

![笔记工作区](images/note-and-agent.png)

在左栏或图谱中打开笔记，中央区域显示阅读/编辑内容。当前笔记的微缩关系图会展示直接出链、反链与待创建概念，帮助判断一篇笔记在整个知识库中的位置。

编辑、新建、重命名和删除不会立即修改磁盘：它们先进入会话改动层。打开“改动与回收站”后，你能看到每个文件的变更类型、diff 和安全状态；确认写回后才真正改动本地文件。

删除的文件会进入 `.knowledge-agent-trash`，保留 30 天。系统按保存的真实删除时间计算到期时间，过期后自动清理。恢复仍需要 Agent 审核，并由用户在面板确认。

## 5. 使用 Note Agent

右栏是知识库专用 Agent，而不是拥有无限系统权限的通用终端。

1. 点击右上角 Agent 设置图标。
2. 选择模型和 Agent 模式；首次连接在线模型时，在单独弹窗填写自己的 API Key。
3. 在输入框提问，按 Enter 或 Send 发送。
4. Agent 可以读取当前知识库、打开笔记、创建草稿、建议链接、生成 MOC、组织内容与展示图谱；真正写回仍必须走草稿和确认步骤。

输入区下方的信息都有实际含义：

- **模型名称**：当前连接的模型或离线状态。
- **Effort**：当前 Agent 模式的推理强度设定。
- **上下文百分比**：根据当前聊天、正在阅读的笔记、手动加入的笔记和 vault 概览估算，并随会话实时变化。
- **文件夹图标**：把当前笔记加入该 Agent 的上下文。
- **加号**：新建一个相互独立的子 Agent 会话。
- **刷新**：清空当前会话，同时保存可回溯快照。
- **回溯**：恢复最近快照及其中的聊天记忆和上下文。

复杂测试库的生成工具仅用于你明确要求“模拟、压力测试或合成测试数据”时。它不是普通对话的隐藏快捷答案；正常知识库任务仍要经过模型选择工具、权限检查和草稿流程。

## 6. 安全边界

App 默认不读取或写入 `.git`、`.obsidian`、`.claude`、`.venv`、`node_modules`、`.env`、`secret`、`token`、密码、账号等敏感目录或文件。安全面板会分别显示允许与排除的文件清单。

API Key 不会写入项目目录，也不应出现在截图、提交、Issue 或公开文档中。桌面端通过本机 App 配置保存 key；安装包不会携带任何用户的 key。

有关安装包、配置位置与发布前检查，继续阅读 [安装与隐私说明](AGENT_INSTALL_AND_PRIVACY.md)。
