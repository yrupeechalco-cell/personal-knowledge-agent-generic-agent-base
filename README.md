# 个人知识库 Agent

个人知识库 Agent 是一款面向长期知识积累的本地优先 Windows 桌面应用。它以普通 Markdown 文件夹为知识库，在不牺牲文件所有权和可读性的前提下，将文件结构、关系图谱、笔记编辑和 AI Agent 放进同一个工作台。

它借鉴 Obsidian 的双链与图谱体验，但不依赖 Obsidian，也不要求把内容迁移到专有格式。已有的 Markdown 或 Obsidian vault 可以直接作为知识库打开；不使用本应用时，文件仍然是普通的 `.md` 文件。

![个人知识库 Agent：全库关系图谱、文件树和 Agent 工作台](docs/images/workspace-graph.png)

## 为长期积累而设计

随着笔记增多，问题往往不在于“有没有写下来”，而在于能否再次找到、理解、整理并安全地继续推进。传统文件夹容易失去横向关联；纯图谱容易变得杂乱；通用聊天工具又不了解真实文件结构，更不应直接修改本机资料。

个人知识库 Agent 将这几个环节连接成一条清晰的工作流：

1. **连接本地文件夹**：选择一个现有 Markdown 文件夹，或在任意磁盘位置新建知识库。
2. **从结构或关系进入内容**：通过层级文件树、筛选、标签、标签页和全库图谱定位笔记。
3. **在上下文中阅读、编辑和思考**：当前笔记、局部关系和 Agent 同时可见，不需要在多个工具之间来回切换。
4. **先审阅，再写回**：所有重要修改先形成草稿、diff 和安全摘要；确认后才会影响真实磁盘文件。

它适合研究者、创作者、开发者、学生，以及任何需要长期沉淀资料、持续重组思路的人。

## 产品特色

### 本地文件是真正的源数据

知识库就是一个普通文件夹。应用读取 Markdown、`[[双链]]`、标签、frontmatter、附件引用和文件夹层级来建立索引，不把笔记锁在数据库或云端账户里。桌面端会记住最近连接的位置，并能监测允许范围内的外部文件变化；当存在未确认草稿时，自动刷新会暂停，避免覆盖正在进行的工作。

默认安全规则会排除 `.git`、`.obsidian`、`.claude`、`.venv`、`node_modules`、`.env`、`secret`、`token`、密码、账号等敏感路径。知识库不会被自动上传。

### 全库关系图谱用于探索，而非装饰

图谱以真实文档链接为关系来源，默认展示整个知识库，而不是只展示当前笔记附近的一小块。它适合在资料规模变大后识别主题簇、孤立笔记和跨领域连接。

- 左键拖动空白区域可在无限感画布中平移，滚轮可围绕指针平滑缩放。
- 节点支持拖拽和轻量物理分布，便于主动整理视野。
- 悬停笔记时，无关节点与连线会自动淡化，直接关联的节点和连线被突出显示。
- 标签会根据实际可读尺寸渐隐或恢复，避免小窗口和大规模图谱出现不可读的文字堆积。
- 图谱浏览位置会被保留；打开其他笔记后返回，不必重新寻找刚才的区域。

### 文件树保留清晰的上下级关系

左侧不是平铺的笔记列表，而是可折叠的真实文件夹层级。它结合筛选、标签、文件夹权重和当前路径，让知识库即使不打开图谱也保持可读。

用户可以在笔记或文件夹上右键新建、重命名、复制、剪切、粘贴和删除；顶部标签页会保留多个已打开的上下文，支持像浏览器一样切换。文件树和关系图谱各自承担不同任务：前者回答“文件存在哪里”，后者回答“它与哪些知识相连”。

### 笔记页面保留局部关系

![笔记阅读、局部关系图与右侧 Note Agent](docs/images/note-and-agent.png)

打开一篇笔记后，中间区域专注于阅读或编辑，局部关系小图持续标出当前笔记的出链、反链和待创建概念。这样既能在大图谱中把握全局，也能在写作时看清一篇笔记所处的局部位置。

未解析概念会以较弱的视觉形式呈现，可直接点击创建为新的 Markdown 草稿；是否将其发展为长期笔记，仍由用户结合上下文判断。

### Note Agent 是受控的知识库操作台

右侧 Note Agent 不是只会回答问题的聊天框，而是围绕当前知识库工作的专用控制台。它可以读取当前笔记与相关上下文，建议链接、生成 MOC、组织内容、打开笔记、筛选图谱、创建草稿，以及请求删除操作。

- Agent 通过明确的 App 工具调用文件能力，不拥有无限制的系统或终端权限。
- 所有涉及文件内容或删除的操作仍然进入草稿与确认流程，模型不能绕过用户的最终确认。
- Agent 任务按会话隔离：可以新建多个编号子 Agent，在一个会话思考或执行时切到另一个会话继续工作。结果始终回到发起它的会话，应用不会因为等待模型而整体不可用。
- 右下角显示当前模型、推理强度和真实上下文估算。上下文进度由聊天记录、当前笔记、手动加入的笔记和知识库概览共同计算，而不是静态装饰。
- 刷新聊天会保留可回溯快照；回溯会恢复最近一次会话内容及其上下文。

应用目前默认支持 DeepSeek V4 Pro / Flash。每位用户在首次使用在线 Agent 时填写自己的 API Key；密钥只保存在本机配置中，不会写进项目、安装包或 GitHub 仓库。

### 每一次改动都可审阅、可恢复

![本机存储与知识库入口](docs/images/local-storage.png)

创建、编辑、重命名和删除不会立即改动磁盘。它们先进入“待写回改动”层，用户可以查看涉及文件、diff 和安全摘要，再确认写回。被删除的文件会进入知识库内的 `.knowledge-agent-trash` 回收站，保留 30 天并按照真实时间自动清理；恢复操作需经过 Agent 审核和用户确认。

这套机制让 Agent 和图谱拖拽删除具备实际操作能力，同时保留对重要资料的控制权与可追溯性。

## 开始使用

1. 在 [GitHub Releases](https://github.com/yrupeechalco-cell/personal-knowledge-agent-generic-agent-base/releases) 下载最新 Windows 安装包。
2. 启动应用，点击左上角文件夹图标，选择“打开已有文件夹”或“选择位置并新建”。
3. 通过文件树或关系图谱打开笔记；首次体验也可以使用 Demo 知识库。
4. 打开右侧 Agent 设置，选择模型与工作模式，并在需要时填写自己的 API Key。
5. 编辑笔记或让 Agent 提出整理方案；在“改动与回收站”检查变更后，确认写回本地文件夹。

浏览器体验入口：[个人知识库 Agent Web](https://personal-knowledge-agent.pages.dev/)。Web 与桌面 App 共用工作台和公开演示库，但浏览器不会获得桌面端的任意磁盘写入、系统安全存储或本机模型配置。

完整操作说明见 [使用指南](docs/USAGE_GUIDE.md)，安装与数据边界见 [隐私说明](docs/AGENT_INSTALL_AND_PRIVACY.md)。

## 当前平台与范围

| 项目 | 当前状态 |
| --- | --- |
| 日用客户端 | Windows 桌面 App（Tauri + React） |
| 数据格式 | 普通 Markdown，兼容常见 Obsidian 双链习惯 |
| 本地文件夹 | 支持打开、新建、读取、草稿确认写回、回收站和文件变化监测 |
| 浏览器入口 | 提供本地预览与用户授权后的文件夹读取；不具备桌面端的任意位置建库能力 |
| 在线模型 | 默认面向 DeepSeek V4 Pro / Flash，用户自备 API Key |
| GitHub 协作 | Git 工作流与安全清单已有基础；GitHub OAuth、实时多人协作和自动云同步仍在后续计划中 |

## 隐私承诺

- 不内置、不收集、不提交用户 API Key。
- 不自动上传真实知识库。
- 公开仓库不包含个人 vault、私有项目记忆或本机配置。
- 任何未来 GitHub 同步都应由用户主动发起，并在提交前确认文件清单与安全状态。

## 开发与构建

```bash
npm install
npm run dev:web       # 浏览器预览
npm run tauri:dev     # Tauri 桌面 App
npm run typecheck
npm test
npm run tauri:build   # Windows 安装包
npm run deploy:web    # 构建、隐私扫描并发布主 Cloudflare Pages 站点
```

## 项目结构

```text
apps/desktop       Tauri 桌面入口：本地文件、设置、模型 key、写回与 Git
apps/web           浏览器预览入口
packages/workspace 共享三栏工作台
packages/core      Markdown、wikilink、标签、图谱与安全清单解析
packages/agent     GenericAgent-inspired 笔记 Agent、工具与权限规则
packages/ui        文件树、编辑器、关系图、微缩图与 Agent console
```
